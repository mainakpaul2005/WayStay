🚀 Full-Stack Project Roadmap (WayStay)
Audience: 2nd-year student building a resume project
Commitment: 10–15 hrs/week
Tech Stack: Spring Boot (backend), MySQL (database), React + Tailwind (frontend), JWT
(auth)

🎯 Project Goal
Deliver a working, deployed web app that demonstrates:
•
•
•
•
•
•

Secure REST API design
Database integration
Authentication & authorization with JWT
CRUD + relationships (users, hotels, bookings, blogs)
Frontend with protected routes and state management
Deployment + documentation

🧩 Core Features
1. User Authentication (Foundation)
Backend (Spring Boot):
•
•

User entity: id, name, email, password (hashed with Bcrypt).

Endpoints:
o
o
o

•

POST /api/auth/register → register user.
POST /api/auth/login → return JWT.
GET /api/users/me → return logged-in user’s profile.

Spring Security + JWT filter for request authentication.

Frontend (React):
•
•
•
•

Pages: /register, /login.
Store JWT in localStorage.
React Context for global auth state.
Protected routes (redirect unauthenticated users).

•

Profile page: display data from /api/users/me.

2. Hotels & Bookings (Core Logic)
Backend (Spring Boot):
•
•
•

Hotel entity: id, name, location, description, price_per_night, image_url.
Booking entity: id, check_in_date, check_out_date, linked to User and Hotel.

Endpoints:
o
o
o
o

GET /api/hotels → list hotels.
GET /api/hotels/{id} → hotel details.
POST /api/bookings (protected) → create booking.
GET /api/my-bookings (protected) → user’s bookings.

Frontend (React):
•

Pages:

/ (home) → list hotels (grid of HotelCards).
/hotels/{id} → hotel detail + booking form.
/my-bookings → user’s bookings.
Components: HotelCard, BookingForm.
o
o
o

•

3. Blog (CRUD Showcase)
Backend (Spring Boot):
•
•

Blog entity: id, title, content, created_at, linked to User.

Endpoints:
o
o
o
o

GET /api/blogs → list posts.
POST /api/blogs (protected) → create post.
PUT /api/blogs/{id} (protected) → edit post (author only).
DELETE /api/blogs/{id} (protected) → delete post (author only).

Frontend (React):
•

Pages:
o
o
o

/blog → list all blogs.
/blog/new → create post (protected).
/blog/edit/{id} → edit post (protected, author only).
